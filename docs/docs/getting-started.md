# Getting Started with AI-BOM: Generate an AI Bill of Materials

AI-BOM is an open-source CLI tool that helps teams understand where AI is used in their software.

By scanning your codebase, it creates an **AI Bill of Materials (AI-BOM)** — a machine-readable list of AI-specific components such as large-language models (LLMs), agent frameworks, prompts, API keys, and other AI building blocks.

Knowing these components helps you manage risks, licensing, and supply-chain obligations.

This tutorial shows how to install AI-BOM, run your first scan, understand the output, and integrate it into your development workflow.

---

## What is an AI-BOM?

An AI Bill of Materials is similar to a Software Bill of Materials (SBOM), but focused specifically on AI-centric dependencies.

AI-BOM scans your code and infrastructure to detect:

- **LLM providers** – e.g., OpenAI, Anthropic, Google Gemini, Ollama  
- **Models and embeddings** – model names passed to libraries like `openai`, `langchain`, or `llama-cpp`  
- **Agent frameworks** – LangChain, CrewAI, AutoGen, LlamaIndex  
- **Prompts and LLM chains** – inline prompt templates and chain definitions  
- **API keys and secrets** – environment variables used to store provider keys  
- **Agents, tools, and containers** – custom agents, tools, and AI container images  

AI-BOM uses multiple scanners and supports output formats including JSON, YAML, SARIF, and CycloneDX.

## Installation

AI-BOM requires **Python ≥ 3.10**.

### Recommended: Install using pipx

```bash
pipx install ai-bom
Alternative: Install inside a virtual environment
python3 -m venv .venv
source .venv/bin/activate
pip install ai-bom
```
 
Verify installation:
```bash
ai-bom --help

```
First Scan
You can scan any project directory:

```bash
ai-bom scan .
```

For this tutorial, we use the built-in demo project:

```bash
ai-bom demo
```
Within seconds, AI-BOM discovers:

LLM providers

Agent frameworks

Model references

Hardcoded API keys

Cloud AI services

Containers and endpoints

Risk scores and flags

Each detected component includes metadata such as location, type, provider, and risk rating.

Understanding the Output

The JSON report generated by AI-BOM includes structured metadata for each component.

Important fields include:

Field	Description
name	Name of the AI component (e.g., openai, claude-3, langchain)
type	Category such as llm-provider, model, agent-framework, tool, or api-key
confidence	Match confidence score (0–1)
risk_score	Risk rating based on heuristics
file	File where the component was discovered
line	Source line number

This helps teams understand AI usage and assess compliance or security risks.

CI/CD Integration

AI-BOM can be integrated into CI pipelines to automatically detect AI components during pull requests.

Example GitHub Actions workflow:

- name: Run AI-BOM scan
  run: ai-bom scan . --fail-on medium

- name: Upload scan report
  uses: actions/upload-artifact@v4
  with:
    name: ai-bom-report
    path: ai-bom-report.json

Useful flags:

--fail-on → Fail pipeline based on risk level

-f cyclonedx → Generate CycloneDX SBOM

-f sarif → Output SARIF for GitHub Code Scanning

This ensures AI usage is continuously monitored.
